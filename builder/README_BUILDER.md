# README for Builder AI

Contents:
- `OMNISCOPE_SPEC.md` — consolidated spec for implementation.
- `ACCEPTANCE_TESTS.md` — tests including Universal Analyst Test.
- `ROADMAP.md` — prioritized milestones and rough estimates.
- `BACKLOG.md` — developer task breakdown with file refs.
- `MANIFEST.json` — machine-readable manifest.
- `UNIVERSAL_ANALYST_FIXTURE.json` — sample fixture for the Universal Analyst Test.

Usage:
1. Use `MANIFEST.json` and `UNIVERSAL_ANALYST_FIXTURE.json` to plan automated build steps.
2. Use `ACCEPTANCE_TESTS.md` to create CI jobs and test harnesses.
3. Inspect referenced repo files (e.g., `backend/app/services/data_cube.py`, `frontend/components/...`) to integrate implementation.
